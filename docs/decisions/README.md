# Architecture decision records

Use an ADR for a decision that changes system boundaries, data ownership, environment topology, provider choice, security model, deployment model, or another choice that is costly to reverse.

1. Copy [`../templates/ADR.md`](../templates/ADR.md) to this directory.
2. Name it `NNNN-short-decision-name.md` using the next four-digit number.
3. Open it as `proposed`, compare realistic options, and identify validation/revisit triggers.
4. Change it to `accepted`, `rejected`, or `superseded` only after the decision is made.
5. Link the ADR from the project plan, delivery tracker, change plan, and pull request where relevant.

Do not rewrite an accepted ADR to hide the old decision. Add a new ADR that supersedes it.
